Setting upload options in InfoLister.

< Back to main page

Step-by-step guide

Setting up Upload in InfoLister is very easy. Here's a step-by-step guide:

  • 1. Make sure you have latest version of InfoLister installed and that you are running Firefox or Nvu (Thunderbird version of InfoLister doesn't support uploading).
  • 2. Select Tools > Extensions. (Extensions Manager window will open.)
  • 3. In that window find InfoLister entry and double-click it. (InfoLister options window will open.)
  • 4. In InfoLister options window select FTP Settings tab. You will see a dialog like this:
  • 5. Fill in the form. You should get your host, user name and password from your hosting provider. In the path field enter the server path to the file you want to upload to including file name (not including the domain name, but including the real path to the file on the server).
  • 6. Click Upload button. The information should be uploaded to the location you selected.

Troubleshooting

  • When I click Upload nothing happens!

    This probably means you're using InfoLister 0.8. It had a typo in its code, which prevented the upload button from working. Try the upload button in main InfoLister window or upgrade to a newer version. If the problem persists, please report it to the author (see below). Be sure to include InfoLister and host application (Firefox or Nvu) versions and any InfoLister-related messages you see in Tools > JavaScript Console.

  • It says it uploaded the file, but it's not on the server!

    This probably means you specified the wrong path. My testing has shown that InfoLister neither complains about nor creates non-existent directories specified in path.
    Also, remember that the publicly accessible files usually are in a www or public_html subdirectory.
    If nothing else helps, open the FTP client that works, navigate to the directory you want to upload the file to and copy the path to InfoLister settings.

  • I get The browser is currently offline message.

    Uncheck File > Work Offline and try again.

  • I get The lookup of a hostname failed, The connection was refused or The operation timed out message.

    Check the value in the Host textbox. Try again later, it may be due to network problems. Also make sure your firewall software doesn't block host application (Firefox or Nvu).

  • I get Login Failure message.

    Check your user name and password. Try logging in to the server with a real FTP client to see if your login information is correct.

  • I get one of these messages FTP CWD Failure, FTP PASV Failure, FTP PWD Failure, FTP List Failure.

    This probably indicates a problem with the FTP server. Try connecting using a real FTP client to see if it works.

  • I get an Unexpected Error message.

    Try creating new Firefox profile. If this helps, something must be wrong with your current configuration or InfoLister settings. Otherwise, please report this problem to the author (see below).
